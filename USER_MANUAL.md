# BoxVisio BI - User Manual (Training Edition)

## 1. Πώς να χρησιμοποιήσεις αυτό το manual
Αυτό το εγχειρίδιο είναι σε μορφή εκπαίδευσης.

Σε κάθε μάθημα θα βρεις:
- στόχο μαθήματος
- απλά βήματα
- μικρή άσκηση ελέγχου

Προτεινόμενος τρόπος:
1. Άνοιξε την αντίστοιχη σελίδα στο tenant portal.
2. Ακολούθησε τα βήματα με τη σειρά.
3. Κάνε την άσκηση και επιβεβαίωσε ότι κατάλαβες τη λειτουργία.

## 2. Κουμπί Βοήθειας μέσα από κάθε κύκλωμα
- Σε κάθε κύκλωμα υπάρχει κουμπί `Βοήθεια`.
- Το κουμπί ανοίγει το `/tenant/manual` στη σωστή ενότητα.
- Παράδειγμα:
  - Παραστατικά Πωλήσεων -> `/tenant/manual#sales-documents`
  - Παραστατικά Αγορών -> `/tenant/manual#purchase-documents`

## 3. Τι σημαίνει το `SoftOne Sync` επάνω δεξιά
- Στο πάνω δεξιά μέρος της οθόνης εμφανίζεται η ένδειξη `SoftOne Sync`.
- Δείχνει την τελευταία ημερομηνία και ώρα που το BI πήρε δεδομένα από το SoftOne του tenant.
- Αν βλέπεις παλιά ώρα:
  - έλεγξε αν έχει ολοκληρωθεί ο συγχρονισμός
  - έλεγξε αν τα KPIs που βλέπεις αντιστοιχούν στην περίοδο που περίμενες
- Αν δείχνει `Δεν υπάρχει ακόμη`, σημαίνει ότι δεν έχει καταγραφεί ακόμη επιτυχής συγχρονισμός για τον tenant.

## 4. Μαθήματα Επιχειρησιακών Κυκλωμάτων

### 3.1 Μάθημα: Παραστατικά Πωλήσεων
- URL: `/tenant/sales-documents`
- Εικόνα εκπαίδευσης:
  - ![Παραστατικά Πωλήσεων](backend/app/static/docs/streams/sales-documents.svg)
- Στόχος:
  - Να εντοπίζεις γρήγορα ποιοι πελάτες/είδη οδηγούν τον τζίρο.
- Βήματα:
  1. Όρισε περίοδο.
  2. Επίλεξε υποκατάστημα.
  3. Χρησιμοποίησε αναζήτηση πελάτη ή είδους.
  4. Δες KPI + πίνακα.
  5. Κατέγραψε συμπέρασμα.
- Άσκηση:
  - Βρες τις 5 υψηλότερες πωλήσεις της εβδομάδας.

### 3.2 Μάθημα: Παραστατικά Αγορών
- URL: `/tenant/purchase-documents`
- Εικόνα εκπαίδευσης:
  - ![Παραστατικά Αγορών](backend/app/static/docs/streams/purchase-documents.svg)
- Στόχος:
  - Να ελέγχεις το κόστος και να προετοιμάζεις διαπραγμάτευση.
- Βήματα:
  1. Φίλτρο σε προμηθευτή + ημερομηνία.
  2. Έλεγχος μεταβολών κόστους.
  3. Σύγκριση προμηθευτών.
  4. Επισήμανση κρίσιμων ειδών.
- Άσκηση:
  - Εντόπισε 3 είδη με μεγαλύτερη αύξηση κόστους.

### 3.3 Μάθημα: Παραστατικά Αποθήκης
- URL: `/tenant/warehouse-documents`
- Εικόνα εκπαίδευσης:
  - ![Παραστατικά Αποθήκης](backend/app/static/docs/streams/warehouse-documents.svg)
- Στόχος:
  - Να ελέγχεις κινήσεις και να προλαβαίνεις ελλείψεις.
- Βήματα:
  1. Φίλτρο σε αποθήκη και τύπο κίνησης.
  2. Επιλογή είδους.
  3. Έλεγχος εισαγωγών/εξαγωγών.
  4. Απόφαση για αναπλήρωση.

### 3.4 Μάθημα: Ταμειακές Συναλλαγές
- URL: `/tenant/cashflow`
- Εικόνα εκπαίδευσης:
  - ![Ταμειακές Συναλλαγές](backend/app/static/docs/streams/cash-transactions.svg)
- Στόχος:
  - Να έχεις καθαρή εικόνα ρευστότητας (cash in/cash out/net cash).
- Βήματα:
  1. Επίλεξε κατηγορία συναλλαγής.
  2. Βάλε φίλτρα ημερομηνίας και λογαριασμού.
  3. Σύγκρινε εισροές/εκροές.
  4. Έλεγξε καθαρή ροή.
  5. Προγραμμάτισε ενέργειες.

### 3.5 Μάθημα: Υπόλοιπα Προμηθευτών
- URL: `/tenant/suppliers`
- Εικόνα εκπαίδευσης:
  - ![Υπόλοιπα Προμηθευτών](backend/app/static/docs/streams/supplier-balances.svg)
- Στόχος:
  - Να μειώνεις overdue υποχρεώσεις με προτεραιοποίηση πληρωμών.
- Βήματα:
  1. Αναζήτησε προμηθευτή.
  2. Δες open/overdue υπόλοιπο.
  3. Δες aging buckets.
  4. Οργάνωσε σειρά πληρωμών.

### 3.6 Μάθημα: Υπόλοιπα Πελατών
- URL: `/tenant/customers`
- Εικόνα εκπαίδευσης:
  - ![Υπόλοιπα Πελατών](backend/app/static/docs/streams/customer-balances.svg)
- Στόχος:
  - Να βελτιώνεις εισπραξιμότητα και να περιορίζεις καθυστερήσεις.
- Βήματα:
  1. Φίλτρο σε πελάτη/περίοδο.
  2. Έλεγχος ληξιπρόθεσμων.
  3. Ανάλυση aging.
  4. Σχέδιο follow-up.

## 5. Μάθημα: Συμφωνίες Προμηθευτών
- URL: `/tenant/supplier-targets`
- Εικόνα εκπαίδευσης:
  - ![Συμφωνίες Προμηθευτών](backend/app/static/docs/streams/supplier-targets.svg)
- Στόχος:
  - Να ορίζεις και να διαχειρίζεσαι συμφωνίες με σαφή και επαναλήψιμο τρόπο.
- Βήματα:
  1. Συμπλήρωσε στόχο, ποσοστό και σταθερό ποσό επιστροφής.
  2. Επίλεξε συμμετέχοντα είδη.
  3. Αποθήκευσε συμφωνία.
  4. Άνοιξε λεπτομέρειες για έλεγχο.
  5. Χρησιμοποίησε `Εκτύπωση` ή `Αποστολή email`.
- Κανόνας κύκλου ζωής:
  - Ληγμένες συμφωνίες γίνονται ανενεργές και δεν επεξεργάζονται.
  - Για νέα περίοδο χρησιμοποίησε `Αντιγραφή`.

## 6. Μαθήματα Αναλύσεων
- Αναλύσεις Πωλήσεων: `/tenant/sales`
- Αναλύσεις Αγορών: `/tenant/purchases`
- Αναλύσεις Αποθέματος: `/tenant/inventory`
- Είδη Αποθήκης: `/tenant/items`
- Λειτουργικά Έξοδα: `/tenant/operating-expenses`

Χρήση:
1. Ξεκίνα από KPI κάρτες.
2. Πήγαινε στα γραφήματα για τάση.
3. Κατέληξε στον πίνακα για λεπτομέρεια.

## 7. Είδη Αποθήκης: πώς διαβάζω τα status
- Στη σελίδα `/tenant/items` κάθε είδος μπορεί να ανήκει σε status όπως:
  - `Ταχυκίνητο`
  - `Κανονική κινητικότητα`
  - `Αργοκίνητο`
- Το `Ενεργό/Ανενεργό` μπορεί να έρχεται:
  - είτε από το SoftOne
  - είτε από τη λογική του BI με βάση την τελευταία πώληση
- Άρα, αν δεις διαφορά σε status μεταξύ tenants, αυτό μπορεί να είναι συνειδητή ρύθμιση του tenant και όχι λάθος.

## 8. Κανόνες SoftOne: τι πρέπει να ξέρεις σαν χρήστης
- Οι κανόνες παραστατικών στο BI έχουν φτιαχτεί ώστε να ακολουθούν τη λογική του SoftOne.
- Τα ονόματα παραστατικών εμφανίζονται με την ονομασία του SoftOne για να μην υπάρχουν μπερδέματα.
- Τα πιστωτικά παραστατικά δεν σημαίνουν πάντα “έξοδο”.
  - Μπορεί να είναι αντιστροφή εσόδου.
  - Γι’ αυτό οι κανόνες ξεχωρίζουν:
    - αν μετράει σε έσοδα
    - αν μετράει σε ποσότητες
    - αν μετράει σε κόστος
    - ποιο είναι το πρόσημο της αξίας
- Αν κάτι φαίνεται “ανάποδο” σε KPI, πρέπει να ελεγχθεί πρώτα ο κανόνας του παραστατικού.

## 9. Κανόνες KPI ανά σειρά παραστατικού
- Το BI μπορεί να έχει ειδικούς κανόνες συμμετοχής KPI ανά σειρά παραστατικού.
- Αυτό δεν αλλάζει τα αποθηκευμένα παραστατικά.
- Αλλάζει μόνο το αν μια σειρά συμμετέχει ή όχι σε συγκεκριμένο KPI, π.χ. στον τζίρο.
- Η λειτουργία αυτή χρησιμοποιείται όταν θέλουμε:
  - να παραμείνει το νέο BI γενικό και σωστό
  - αλλά να ταιριάξει και με αποδεδειγμένη επιχειρησιακή λογική ενός tenant
- Άρα:
  - τα raw δεδομένα μένουν μέσα
  - οι KPI κανόνες αποφασίζουν πώς θα διαβαστούν

## 10. Συχνά λάθη και γρήγορες λύσεις
- Δεν βλέπω δεδομένα:
  - Έλεγξε ημερομηνία, υποκατάστημα, φίλτρα επιλογών.
- Βλέπω παλιά δεδομένα:
  - Κοίτα πρώτα το `Τελευταίος συγχρονισμός` επάνω δεξιά.
- Δεν βρίσκω συμφωνία:
  - Έλεγξε έτος στόχου και αναζήτηση ονόματος.
- Δεν επιτρέπει επεξεργασία συμφωνίας:
  - Πιθανότατα είναι ανενεργή (λήξη περιόδου). Χρησιμοποίησε `Αντιγραφή`.
- Το status είδους δεν μου φαίνεται σωστό:
  - Έλεγξε αν ο tenant παίρνει το `Ενεργό/Ανενεργό` από SoftOne ή από τελευταία πώληση.

## 11. Συντόμευση εκπαίδευσης για νέα μέλη ομάδας (30 λεπτά)
1. 10 λεπτά: Παραστατικά Πωλήσεων + Αγορών.
2. 10 λεπτά: Υπόλοιπα Προμηθευτών + Πελατών.
3. 10 λεπτά: Συμφωνίες Προμηθευτών + Εκτύπωση/Email.
